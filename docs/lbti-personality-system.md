   结果页面：

   **三套人格同属一个测试结果，如何在前端优雅地展示而不显冗杂？**

   核心矛盾是：**信息量巨大**（三套角色各含代号、中文名、判词，甚至还有 Badge 和详细描述），但用户注意力有限，且必须保留 **“截图分享”** 的传播属性。

   以下提供三套成熟的前端交互方案，从简到繁，你可以根据开发资源选择。

   ------

   ### 🧩 方案一：书签式/卡片叠层（推荐，最符合 SBTI 传播调性）

   这是最像“塔罗牌/身份卡”的展示方式，适合移动端竖屏滑阅，**一次只看一张卡，滑动切换人格面**。

   **1. 页面结构**

   - **顶部固定区**：测试主标题 + 用户头像/昵称 + 一句引导语（例如：“你在爱情里的三重身份”）
   - **中央卡片区**：一张大卡片，通过左右滑动或点击标签切换“正面/反面/动物面”
   - **底部操作区**：保存卡片、分享结果、重新测试

   **2. 卡片切换交互**

   - 使用 **三个小圆点指示器** 或 **三个小图标**（😈自嘲 / 🐾动物 / 🌸甜心）标明当前所在面。
   - 手指左右滑动 → 卡片翻转变换（可加 3D flip 动画或平滑淡入淡出）。
   - 每一面卡片内布局一致：代号在上、中文名居中、判词在下，背景色区分（如自嘲面灰黑、动物面暖棕、甜心面粉金）。

   **3. 视觉隐喻**

   - 把人格比作 **“一枚硬币的三个面”** 或 **“一面镜子的三种倒影”**。
   - 加载文案：“转动卡片，看见你的另一面”。

   **4. 分享图生成**

   - 提供三个独立的分享按钮：“分享自嘲面”、“分享动物面”、“分享甜心面”。
   - 也可生成一张 **长图拼接**，包含三张小卡片，供用户发朋友圈九宫格。

### 📱 最终推荐：方案一（书签式滑动）+ 底部三个小标签

这是经过多个心理测试 H5 验证的最优解：

> **页面示意**
> ——————————————————
> 🌙 你的爱情人格
> “原来你有三副面孔”
> ——————————————————
> [ 🖤 ] [ 🐾 ] [ 🌸 ] ← 标签切换
> ——————————————————
> | |

> | 偏爱独奏者 |
> | |

> | 是只对一个人开放 |
> | 的独奏会...” |
> | |
> ——————————————————
> [ 查看完整档案 ] [ 分享此面 ]
> ——————————————————

- 点击标签或左右滑动，卡片内容、背景色、标签高亮同步切换。
- “查看完整档案”以抽屉弹窗形式展示该人格的 Badge、核心画像、发光点与风险点。
- 分享时自动带上当前选中面的卡片截图与文案。
