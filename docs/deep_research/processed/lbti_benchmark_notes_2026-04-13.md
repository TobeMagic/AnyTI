# LBTI Benchmark Notes

**Date:** 2026-04-13  
**Scope:** Rebuild the `LBTI` lane by combining the structural strengths of `16Personalities` with the viral language and share mechanics of `SBTI / sbti.one`, while going deeper on relationship-specific modeling.

## Sources Used

- `16Personalities` free test page: https://www.16personalities.com/free-personality-test
- `16Personalities` framework article: https://www.16personalities.com/articles/our-theory
- `sbti.one` home: https://sbti.one/en/
- `sbti.one` page source inspection on 2026-04-13

## What 16Personalities Does Better

### 1. It feels like an official product, not a throwaway page

The page architecture is orderly and trust-heavy:
- strong hero
- explicit navigation
- visible scale and global reach
- method explanation
- result system that looks internally coherent

This matters because users will tolerate playful labels more easily if the shell looks like a real product.

### 2. It explains why the model exists

The `Our Framework` page makes a very important move: it openly frames itself as a hybrid of type readability and trait-based measurement. The useful product lesson is not “copy MBTI letters,” but:

- define independent dimensions clearly
- explain why type labels still exist on top of continuous scores
- make the system sound considered, not mystical

### 3. It has layered information depth

`16P` is good at:
- short hook first
- deeper explanation second
- type group logic third

For `LBTI`, this translates to:
- screenshot label
- one-line hit
- short explanation
- deeper dimension read

## What sbti.one Does Better

### 1. It leads with attitude

The home headline is aggressive and simple. It does not over-explain. It creates immediate curiosity and a social pose.

### 2. It is built for multiple editions

The live site presents editions and variants from the start. That matches the matrix strategy better than a single isolated test.

### 3. It exposes its dimensional ambition

On 2026-04-13 source inspection, `sbti.one` showed:
- `15` dimensions
- grouped into `5` models
- result page structure with `Quick Read`, `Dimension Scores`, and poster output

This is important because it proves the viral format does not need to stay shallow. A product can be meme-friendly and still have a multi-model backend.

### 4. It is poster-native and mobile-legible

The site explicitly tells users how to save the poster on mobile. That is a strong reminder that the real product is not just the test itself, but the exported identity object.

## Design Rules Derived For LBTI

### Structure

- Home page should be a channel gateway, but love gets hero placement first
- Category page should feel like a themed editorial channel, not a folder
- Test page should explain the method before the quiz starts
- Result page should be readable as a long mobile story, not only as desktop cards

### Content

- Use `16` visible archetypes plus `1` hidden archetype
- Archetype names should behave like social labels, not therapy labels
- Each archetype needs:
  - name
  - badge
  - one-line hit
  - short rationale
  - strength / stress / repair guidance

### Science Layer

`LBTI` should go deeper than the generic “attachment + conflict” framing. The current implemented `6`-model design is:

1. `attunement`
2. `signal`
3. `repair`
4. `boundary`
5. `certainty`
6. `retreat`

This is stronger than a simple love-style MBTI clone because it separately models:
- wanting closeness
- showing closeness
- repairing conflict
- protecting space
- needing definition
- handling exit / withdrawal

## Naming Rules Derived For Virality

- Prefer role words, job words, device words, or system words
- Let the label be a little sharp and a little embarrassing
- Make the user think of one real-life scene immediately
- Frontstage language should be human and social
- Backstage model can remain structured and multi-dimensional

Examples that fit this rule:
- `浪漫审计员`
- `恋爱项目经理`
- `冷战拆弹手`
- `关系气象台`

## Final Product Direction

`LBTI` should not become “more academic SBTI.”  
It should become:

- as legible as `16P`
- as screenshot-ready as `SBTI`
- structurally deeper on relationship behavior than either reference

That means the winning combination is:

- official-site shell
- viral naming
- multi-model scoring
- mobile-first result readability
- poster-first sharing
