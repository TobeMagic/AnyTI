# BTI Matrix Platform

## What This Is

一个以 BTI 风格人格测试为核心的矩阵化静态前端平台，先从 `WBTI` 职场行为类型测试切入，逐步扩展到恋爱、消费、学习、健身、旅游等多个测试方向。产品采用“平台首页 → 类别主页 → 具体测试页”的分层信息架构，每个类别保留独立主题色与统一交互骨架；核心不是单个测试爆发，而是通过共用引擎、独立数据、站内互荐与主站聚合，沉淀一个可持续扩张的低成本测试矩阵。

## Core Value

用一套可复用的静态测试引擎，持续低成本孵化多个高传播测试，并把流量沉淀回自有矩阵资产。

## Context

项目定位不是“做一个测试”，而是构建一个 BTI 测试矩阵平台。每个测试面向不同生活场景与传播节点：职场、恋爱、消费、学习、健身、旅游；单个测试可借势节日或季节节点获取脉冲流量，矩阵结构则负责承接与循环流量。

技术路线强调零成本、零运维、零复杂度：纯静态前端部署到 GitHub Pages；代码一套，数据分离；目录层面由 `shared/` 放通用引擎，每个测试子目录维护 `questions.json`、`personalities.json`、`meta.json` 三份核心数据文件。这样既利于快速复制，也利于后续 SEO 聚合。

科学性设计方向已经明确：每个测试应采用 4-5 个连续谱维度，不使用简单二选一打分；每道题可对多个维度赋权；结果匹配采用用户得分向量与人格向量之间的余弦相似度；每个测试再设计 1-2 个隐藏人格，通过极端值或特殊答题路径触发，以制造传播彩蛋与稀缺感。

产品传播机制也已确定为必须内置：结果页需要 Canvas 海报导出能力，海报上有人格名、核心描述、二维码，并适配朋友圈/小红书内容尺寸；结果页底部要推荐其他测试，把单次测试消费转成矩阵级浏览。人格标签和结果短文案不再走学术型命名，而要偏“抽象、好笑、能截图、能接龙”的社交语言，以提高转发与评论区自传播概率。

现阶段优先级清晰：先完成 `WBTI` 作为模板，再复用到 `LBTI`、`STBI`、`SBTI`、`FBTI`、`TBTI`。前端方法论上，后续 UI 设计合同应结合 `gsd-ui-phase`、`ui-ux-pro-max`、`ckm:ui-styling` 的方式来锁定页面层级、色彩、组件与交互约束，而不是边做边拼。对角色体系的设计，则需要专项研究 `SBTI` 的爆火机制，吸收其传播优点，同时在结构化、可复用性和矩阵扩展能力上做得更强。

## Constraints

- **Tech stack**: 纯静态前端 + GitHub Pages — 用户明确要求零运维、零托管成本。
- **Architecture**: 引擎共用、数据分离 — 后续扩测试时必须保持边际成本接近零。
- **IA**: 首页 → 类别主页 → 测试页 — 这是已锁定的前端层级，而不是可选项。
- **UI system**: 不同测试类别必须有不同主题色，但共享统一布局骨架与交互模式 — 例如恋爱类可以使用粉红色系。
- **Scope**: 当前优先锁骨架与设计约束，不直接展开所有类别的最终视觉稿和全部角色文案。
- **Launch strategy**: `WBTI` 优先 — 必须先把单个模板测试全链路跑通。
- **Sharing**: 结果海报为必选项 — 传播能力不是附加项，而是产品内建能力。
- **SEO / IA**: 目录与信息架构需支持后续矩阵化聚合 — 不能只为单一测试临时拼装。
- **Content tone**: 结果标签和短描述必须优先服务传播与“被说中”的感觉，而不是追求心理学术语的正式感。
